# Plan: 06-01 - Webhook Infrastructure

**Goal:** Implement the `strava-webhook` Edge Function for real-time activity ingestion.

## Tasks
- [ ] **Task 1: Scaffold Function:** Create `supabase/functions/strava-webhook/index.ts`.
- [ ] **Task 2: Handshake Logic:** Implement verification of `hub.mode`, `hub.verify_token`, and return `hub.challenge` for GET requests.
- [ ] **Task 3: Event Handler POST:**
    - [ ] Acknowledge with 200 OK immediately.
    - [ ] Route `create`, `update`, and `delete` events for `object_type: activity`.
    - [ ] Implement background processing using `EdgeRuntime.waitUntil`.
- [ ] **Task 4: Sync Logic:**
    - [ ] **Create/Update:** Refresh OAuth token -> GET `DetailedActivity` -> GET `ActivityZones` -> Upsert to `activities`.
    - [ ] **Delete:** Hard delete activity from `activities` table.
- [ ] **Task 5: DB Updates:** Verify RLS for `service_role` can perform deletes and upserts on `activities`.

## Verification
- [ ] **Unit:** Test handshake logic (GET /strava-webhook?hub.mode=subscribe&hub.verify_token=...).
- [ ] **Integration:** Mock a POST `create` event and verify the activity is created in the DB with full metrics and zones.
