# Milestone v1.2: Weekly Planning Intelligence

**Status:** ✅ SHIPPED 2026-03-13
**Phases:** 9-11
**Total Plans:** 10

## Overview

Turned `Ukeplan` into the primary AI weekly planning surface so recommendations now reflect long-term week intent, admin coaching philosophy, and athlete day-level constraints without silently overwriting manual edits.

## Phases

### Phase 9: Ukeplan Ownership

**Goal**: Make `Ukeplan` the single tactical weekly-planning surface while `Treningsplan` remains the source of weekly intent.  
**Depends on**: None  
**Plans**: 2 plans

Plans:

- [x] 09-01: Move weekly AI generation into `Ukeplan`
- [x] 09-02: Hand off weekly intent from `Treningsplan`

**Details:**
- `Ukeplan` gained its own AI setup and generation flow.
- Returned structured plans are remapped onto the focused week and persisted through the existing workout-entry model.
- `Treningsplan` was reduced to read-only weekly intent with an explicit handoff into `Ukeplan`.

### Phase 10: Recommendation Context

**Goal**: Feed weekly recommendations with block-level targets plus the existing admin coaching philosophy from `coach_philosophy_documents`.  
**Depends on**: None  
**Plans**: 4 plans

Plans:

- [x] 10-01: Normalize selected-week recommendation context
- [x] 10-02: Prioritize selected-week context in the secure coach prompt
- [x] 10-03: Allow AI planning for any visible `Ukeplan` week
- [x] 10-04: Enforce selected-week AI directives server-side

**Details:**
- Selected week type, mileage target, dates, and notes became explicit payload contract fields.
- Prompt precedence now treats selected-week intent as primary while admin philosophy acts as a guardrailed coaching layer.
- AI planning no longer defaults to the first visible week.
- Server-side enforcement prevents silent taper and mileage drift without explanation.

### Phase 11: Constraint-Aware Weekly Plans

**Goal**: Make generated weeks respect athlete day-by-day constraints, explain tradeoffs, and protect manual edits.  
**Depends on**: None  
**Plans**: 4 plans

Plans:

- [x] 11-01: Capture weekly day constraints in `Ukeplan`
- [x] 11-02: Make the secure coach path explain constraint tradeoffs
- [x] 11-03: Protect manual edits with review-first regeneration
- [x] 11-04: Consolidate verification for the full phase

**Details:**
- `Ukeplan` now captures long-run day, quality-day, commute-day, and double-threshold preferences.
- The edge function reasons over constraints and explains relaxed preferences in coaching language.
- Manual edits persist as protected days and regeneration pauses for review before replacement.
- Phase verification consolidated payload, edge-function, hook, and page-level coverage.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Weekly AI generation belongs in `Ukeplan`, while `Treningsplan` remains the source of weekly intent.
- Selected-week intent outranks philosophy preferences unless a red-line rule must override it.
- Constraint handling must be explainable and review-first rather than silently destructive.

**Issues Resolved:**

- Removed the split ownership between long-term planning and weekly AI generation.
- Removed hidden first-visible-week coupling from weekly AI requests.
- Closed silent drift risk where target mileage or taper intent could be lost in weekly generation.
- Added a persistent protected-day model so manual edits survive regeneration unless the user explicitly replaces them.

**Issues Deferred:**

- Live authenticated Playwright coverage still depends on local auth setup and provider behavior.
- Two manual UX checks remain open around setup-card compactness and preserve-vs-replace clarity.
- Phase 09 Nyquist validation remains partial and should be cleaned up separately.

- Phase summary files still do not expose structured `requirements-completed` frontmatter for stronger audit automation.
