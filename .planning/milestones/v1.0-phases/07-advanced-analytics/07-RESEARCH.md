# Research: Phase 7 - Advanced Analytics

## Standard Stack

- **Math & Statistics:** `simple-statistics` (Recommended). If adding a dependency is blocked, use a robust hand-rolled linear regression utility that provides `slope`, `intercept`, and `rSquared`.
- **Visualization:** `recharts` (Locked). Specifically `ScatterChart`, `Scatter`, `Line`, `Cell`, and `ComposedChart`.
- **Date Handling:** Native `Date` objects combined with existing `compute.js` helpers. Use ISO strings for DB keys.

## Architecture Patterns

### 1. Data Processing Pipeline (in `compute.js`)
1. **Filter:** Activities > 20 mins, HR > 0, Type = 'Run'.
2. **Normalize:**
    - Calculate **Efficiency** = `Speed (km/h) / Avg HR`.
    - Handle **GAP** (Grade Adjusted Pace):
        - If Strava doesn't provide it (default), calculate an estimated **Mean Grade GAP** using the Minetti Formula (5th-order polynomial) where `grade = elevation_gain / distance`.
        - *Crucial:* For aerobic efficiency, the best data comes from runs with < 1% mean grade. Tag these as "High Quality" points.
3. **Regress:**
    - Input: `[daysSinceStart, efficiencyValue]`
    - Output: `{ slope, intercept, rSquared }`

### 2. ScatterChart with Trend Line
- Use `ComposedChart` to overlay a `Line` (the regression) on top of `Scatter` points.
- The `Line` data is generated by taking the first and last dates of the window and calculating `y = slope * x + intercept`.

### 3. Intensity-Based Coloring
- Calculate **Intensity Score** (0.0 to 1.0) based on % of time in Zones 3-5.
- Use Recharts `<Cell />` within `<Scatter />` to apply a color gradient:
    - Low Intensity (Z1-Z2): Emerald/Blue.
    - High Intensity (Z4-Z5): Amber/Red.

## Don't Hand-Roll

- **Regression Math:** Do not write custom matrix math. Use `simple-statistics` or a verified least-squares snippet.
- **Minetti Formula:** Use the standard 2002 coefficients: `Cr = 155.4i^5 - 30.4i^4 - 43.3i^3 + 46.3i^2 + 19.5i + 3.6`.
- **Date Arithmetic:** Avoid `date + (7 * 24...)`. Use `new Date()` with `setUTCDate` to avoid timezone/DST bugs.

## Common Pitfalls

- **Division by Zero:** Always check `avg_heartrate > 0` and `moving_time > 0` before computing speed or efficiency.
- **Noise Sensitivity:** A single "race effort" or "very hilly run" will spike efficiency. 
    - *Fix:* Use `rSquared` to show a "Confidence Score" for the trend.
    - *Fix:* Exclude runs with extreme grades (> 5% average) from the regression but keep them as points.
- **Recharts Performance:** Rendering 150+ scatter points with individual `<Cell />` components can be slow. Ensure data is memoized.

## Code Examples

### Linear Regression (Simple Statistics)
```javascript
import { linearRegression } from 'simple-statistics';

const points = data.map(d => [d.days, d.efficiency]);
const { m, b } = linearRegression(points);
// Trend Line: y = mx + b
```

### Minetti GAP Factor
```javascript
function getMinettiFactor(grade) {
  const i = grade; // as decimal, e.g. 0.05
  const cost = (155.4 * i**5) - (30.4 * i**4) - (43.3 * i**3) + (46.3 * i**2) + (19.5 * i) + 3.6;
  return cost / 3.6; // Factor relative to flat ground
}
// Adjusted Speed = Actual Speed * Factor
```

### Efficiency Gains %
```javascript
const initialEff = slope * firstDay + intercept;
const finalEff = slope * lastDay + intercept;
const gainPct = ((finalEff - initialEff) / initialEff) * 100;
```
