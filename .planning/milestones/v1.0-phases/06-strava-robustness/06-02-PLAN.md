# Plan: 06-02 - History Pagination Sync

**Goal:** Upgrade `strava-sync` to support stable, full-history pagination using `before` timestamp parameters.

## Tasks
- [ ] **Task 1: Pagination Loop Implementation:**
    - [ ] Update `strava-sync` with a `while` loop using the `before` parameter.
    - [ ] Fetch in 200-item chunks.
    - [ ] Record the timestamp of the oldest activity in each chunk as the next `before` value.
- [ ] **Task 2: Rate Limit Throttling:**
    - [ ] Implement a 1-second delay between pages to ensure stability.
- [ ] **Task 3: Conflict Optimization:**
    - [ ] Skip activities already in the DB during history sync (check `strava_id`).
- [ ] **Task 4: Selective Fetching:**
    - [ ] Only fetch `SummaryActivity` (the list response) for full sync; reserve `DetailedActivity` for new/recent data.

## Verification
- [ ] **Integration Test:** Trigger a sync for a user with >200 activities; verify that the loop continues beyond the first page and all activities are imported without gaps.
